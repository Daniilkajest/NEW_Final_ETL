## Описание проекта
Данный проект представляет собой реализацию полноценного ETL-процесса (Extract, Transform, Load) с использованием современного стека технологий. Цель проекта — настроить репликацию сырых данных из нереляционной базы данных (MongoDB) в реляционное хранилище (PostgreSQL), произвести их очистку, трансформацию и построить аналитические витрины данных (Data Marts) с помощью оркестратора Apache Airflow.

## Стек технологий
* **Источники данных:** MongoDB (v6.0)
* **Хранилище (DWH):** PostgreSQL (v13)
* **Оркестрация:** Apache Airflow (v2.8.1)
* **Язык разработки:** Python 3
* **Инфраструктура:** Docker, Docker Compose

## Архитектура решения и структура репозитория
* `dags/` — директория с DAG-файлами для Airflow (`mongo_to_pg_dag.py`).
* `generator/` — Python-скрипт для генерации моковых данных (сессии пользователей и тикеты поддержки) в MongoDB с использованием библиотеки `Faker`.
* `init_sql/` — DDL-скрипты для инициализации хранилища PostgreSQL (схемы `ods` и `marts`, создание таблиц и партиций).
* `docker-compose.yml` — конфигурация для локального развертывания всей инфраструктуры.

## Описание ETL-пайплайна
Пайплайн `mongodb_to_postgres_etl` реализован в Airflow и состоит из следующих логических этапов:

1. **Extract (Извлечение):** * Подключение к MongoDB через `pymongo`.
   * Извлечение документов из коллекций `UserSessions`.
2. **Transform (Трансформация и очистка):**
   * Приведение строковых дат стандарта ISO 8601 из MongoDB к типу `TIMESTAMP` для корректной работы в PostgreSQL.
   * Конвертация массивов MongoDB (например, списков посещенных страниц) в тип данных `ARRAY` в PostgreSQL.
3. **Load (Загрузка и дедупликация):**
   * Данные загружаются в слой `ods` (Operational Data Store).
   * Реализован механизм UPSERT (`INSERT ... ON CONFLICT DO UPDATE`), гарантирующий идемпотентность пайплайна и отсутствие дубликатов при перезапусках.
   * Таблица `ods.user_sessions` **партиционирована** по месяцам (ключ `start_time`), что обеспечивает оптимальную работу аналитических запросов.
4. **Data Marts (Построение витрин):**
   * Выполнение SQL-трансформаций средствами `PostgresOperator` для формирования аналитических витрин в схеме `marts`.

## Аналитические витрины
В рамках проекта реализованы две витрины:
1. **`marts.dm_user_activity` (Активность пользователей):** Агрегирует данные о сессиях (общее количество, суммарное время на сайте в минутах) и определяет наиболее часто используемое устройство (через агрегатную функцию `MODE()`).
2. **`marts.dm_popular_actions` (Популярные действия):** Разворачивает массив пользовательских действий (`UNNEST`) и подсчитывает частоту каждого события для поведенческой аналитики.

## Инструкция по запуску (Local Environment)

1. **Запуск инфраструктуры:**
   ```bash
   docker compose up -d
2. **Генерация исходных данных:**
   Создайте виртуальное окружение, установите зависимости и запустите скрипт:
   ```bash
   uv venv
   source .venv/Scripts/activate
   uv pip install -r generator/requirements.txt
   python generator/generate_mongo_data.py


   3. **Настройка Airflow:**
   * Перейдите в веб-интерфейс `http://localhost:8080` (логин/пароль: `admin` / `admin`).
   * В верхнем меню перейдите в **Admin -> Connections**.
   * Нажмите **+** (Add a new record) и создайте подключение:
     * **Connection Id:** `postgres_default`
     * **Connection Type:** `Postgres`
     * **Host:** `postgres`
     * **Schema, Login, Password:** `airflow`
     * **Port:** `5432`

4. **Запуск ETL:**
   * На главной странице снимите с паузы (Unpause) DAG `mongodb_to_postgres_etl`.
   * Нажмите **Trigger DAG** (кнопка Play) и дождитесь успешного (Success) выполнения всех этапов.
